# Admin Guide

Welcome to the **Anave Decals Management** Admin Dashboard guide. This comprehensive documentation is designed to help administrators navigate the platform, manage products, fulfill orders, and interpret analytics.

> [IMPORTANT] > **Accessing the Dashboard**: Before proceeding, ensure you are logged in.
> [**Click here to access the Admin Dashboard**](https://hobbyistdecals.in/admin-dashboard)

---

## Dashboard

The **Dashboard** is your command center, providing a real-time snapshot of your store's performance. Upon logging in, you are greeted with key metrics and quick access actions.

### Key Metrics

At the top of the dashboard, you will find four primary cards displaying vital statistics:

- **Total Revenue**: The total earnings generated by the platform. It shows a percentage indicator (e.g., `+12.5%`) comparing performance to the previous period.
- **Total Orders**: The cumulative number of orders received. It also highlights the number of **pending** orders that need attention.
- **Products**: The total count of active products. Crucially, it alerts you to **low stock** items (e.g., `4 low stock`), ensuring you can Restock timely.
- **Customers**: The total number of registered customers on the platform, with recent growth stats.

### Quick Actions

To speed up your workflow, the **Quick Actions** section provided colorful, easy-access buttons for common tasks:

- **New Product** (Blue): Immediately jump to the "Add Product" screen.
- **View Orders** (Purple): Go directly to the full order list.
- **Customers** (Pink): Access the customer management directory.
- **Analytics** (Orange): View detailed sales and performance reports.

### Activity & Alerts

#### Recent Orders

On the left, you see a list of the latest 5 orders with pending or active statuses. Each row displays:

- **Order ID**: Unique identifier (e.g., `ORD-176519...`).
- **Customer**: Name of the buyer.
- **Amount**: Total value of the order.
- **Status Badge**: Color-coded status (e.g., `Cancelled` in red, `Delivered` in green, `Shipped` in teal, `Order Placed` in blue).

#### Top Products

On the right, the **Top Products** card highlights your best-selling items. It ranks them by sales volume and revenue, giving you quick insight into what's trending.

#### Low Stock Alert

A prominent **Low Stock Alert** banner appears if items are running low.

- It displays the count of products needing attention (e.g., "You have 4 products with low inventory").
- Click the **View Inventory** button to jump directly to the filtered stock list for restocking.

## Track Order

The **Track Shipment** page (`/admin-dashboard/orders/track`) allows for quick status lookups, ideal for customer support queries.

### How to Search

- **Order ID**: Enter the numeric ID (e.g., `123`).
- **Order Code**: Enter the full code (e.g., `ORD-2024-001`).

### Results Display

- **Active Tracking**: If the order is shipped, you will see real-time updates (Current Location, Expected Delivery).
- **No Data Warning**: If the order hasn't been shipped yet, a "No Tracking Data" message appears, prompting you to create the shipment in the Orders panel first.

## Products

The **Add New Product** flow is a multi-step wizard designed to capture all product attributes.

### Add Product Workflow

#### Step 1: Product Details

1.  **Basic Information**:
    - **Product Name**: Main title of the item.
    - **SKU**: Unique code (e.g., `DECAL-001`). Optional.
    - **Price (USD)**: Base selling price.
    - **Discount (%)**: Optional percentage off (shows as a badge).
    - **Short & Detailed Definitions**: Use the rich text editor to describe the product.
2.  **Category & Availability**:
    - **Category**: Select from existing list (e.g., "Military", "Automotive").
    - **Stock**: Enter quantity and check **In Stock** status.
3.  **Product Tag**: Apply special badges like `featured`, `premium`, `new`, or `bestseller` to highlight items in the store.
4.  **Product Image**: Upload the main display photo.
5.  Click **Next: Variants** to proceed.

#### Step 2: Variants & Configuration

This section allows you to offer customizations. For each variant type, you can use **Quick Select** buttons or type manually, and assign an **Additional Price** if needed.

1.  **Scales**: Define sizes (e.g., `1/32`, `1/64`).
2.  **Materials**: Choose decal types (e.g., `Waterslide`, `Vinyl`, `Glossy`).
3.  **Variations**: Add special options like `Premium` or `Limited Edition`.
    - _Example_: Use "Additional Price" to charge $2.00 extra for a "Premium" variation.
4.  **Quantity Discounts**:
    - Set up tier-based pricing (e.g., Buy 3-4 items, get 10% off).
    - Leave "Max Qty" empty for "X or more" logic.
5.  Click **Create Product** to publish.

#### Step 3: Loyalty Points Configuration

Rewarding customers is built into the product flow.

- **Base Rate**: Shows the default earning rate (e.g., $1 = 1 Point).
- **Active Campaigns**: View special multipliers like "100X Points (summer)".
- **Action**: Click **Add Loyalty Points Campaign** to boost rewards for this specific item.

### Manage Products

The **Edit Products** page (`/admin-dashboard/add-product/list`) provides a card-based view of your entire inventory.

**Product Cards**:
Each card displays key info at a glance:

- **Thumbnail**: The main product image.
- **Badges**: Tags like `FEATURED` or `PREMIUM` in the corner.
- **Details**: Name, SKU, and Price Range (e.g., `$9.98 - 19.98`).
- **Inventory**: Stock count (e.g., `99 units`) and status (`In Stock`).
- **Date**: Last updated or creation date.

**Actions**:

- **Search**: Use the global search bar to find products by name or SKU.
- **Edit**: Opens the full editor to update details.
- **Variants**: Quick access to manage variants (Purple button).
- **Delete**: Remove the product permanently (Red trash icon).

## Orders

The **Orders Management** dashboard (`/admin-dashboard/orders/all`) is your command center for processing customer purchases.

### Overview

#### 1. Performance Stats

At the top, you'll see real-time metrics:

- **Total Orders**: Count of all time orders.
- **Total Revenue**: Sum of all _paid_ orders.
- **Processing**: Number of active orders needing attention.
- **Delivered**: Successfully completed shipments.

#### 2. Search & Filters

- **Search Bar**: Instantly find an order by **Order Code**, **Customer Name**, or **Email**.
- **Filters**: Dropdowns to view specific segments like "Pending Payment" or "Shipped".

### Order Details

Each order is displayed as a detailed card containing:

- **Header**: Order ID, Timestamp, and Status badges.
- **Customer Info**: Name, Email, Billing, and Shipping addresses.
- **Payment Details**: Status (`Paid`/`Unpaid`) and Method.
- **Items**: List of purchased products with quantities and variant details.
- **Order Notes**: Special instructions from the customer (highlighted in yellow).

### Actions & Workflow

Manage the lifecycle of an order using the action buttons:

1.  **Change Status**: Manually update the status (e.g., move from `Pending` -> `Processing`).
2.  **Ship via Shiprocket**:
    - Click **Ship to Shiprocket** to push order details to your logistics partner.
    - Once synced, the button changes to **View Shipment** and displays the **Tracking ID**.
3.  **Cancel**: Use the red button to cancel invalid orders.

## Shipments

The **Shipments** module is your logistics hub, integrated with providers like Shiprocket.

### 1. All Shipments

View a master list of all packages (`/admin-dashboard/shipments/all`).

- **Tabs**: Filter easily by status: `All`, `In Transit`, `Out for Delivery`, `Delivered`.
- **Columns**: Provider, Tracking ID, AWB Code, and Courier.

### 2. Pending Shipments

This critical section (`/admin-dashboard/shipments/pending`) alerts you to orders that need attention.

- **Status**: Orders here typically show **Pending AWB**.
- **Action**: Click **View & Assign** to manually trigger the AWB assignment if the automatic sync failed or is pending.

### 3. AWB Assignments

Track statistics for backend assignment success rates (`/admin-dashboard/shipments/awb`).

- **Stats Board**: Total Assignments, Successful, and Failed.

### 4. Shipment Details

Clicking **View Details** on any shipment opens a granular view.

> **AWB Not Assigned**: If the shipment is created but lacks a label, you will see a yellow warning banner. Click the **Assign AWB Now** button to generate the shipping label and complete the booking immediately.

## Manual Shipping

The **Manual Shipping** module (`/admin-dashboard/manual-shipping`) gives you granular control over non-integrated logistics, allowing you to manage shipments and rates manually.

### 1. Dashboard

The **All Manual Shipments** dashboard is the central hub for tracking manual orders (`/admin-dashboard/manual-shipping/list`).

- **Filters**:

  - **Search**: By Order Code or Tracking Number.
  - **Status**: Filter by specific stages like `Order Placed`, `In Design`, `Processed`, `Shipped`, etc.
  - **Date**: View shipments within a specific `Start Date` and `End Date`.

- **Shipment List**:
  - **Order Code**: Unique identifier.
  - **Customer**: Name and email.
  - **Destination**: Area settings and Country.
  - **Status**: Visual badges for Payment (e.g., `Paid`) and Order Status.
  - **Tracking**: Shows assigned tracking number or "Not assigned".
  - **Shipping Cost**: The calculated cost for the shipment.

### 2. Manage Rates

Configure your shipping bands and costs in the **Shipping Rates** section (`/admin-dashboard/manual-shipping/rates`).

#### Overview

- **Search**: Find rates by Country or Location name.
- **Filters**: Refine by **Location**, **Location Type**, or **Status**.
- **Actions**:
  - **Add Rate**: Manually create a new weight-based rate.
  - **Bulk Tools**: `Download Template` and `Upload Excel` for mass updates.

#### Rate Table

View all active and inactive rates with details:

- **Location**: The zone or country (e.g., "Africa zone").
- **Weight Band**: The defining weight range (e.g., `0.05 kg`).
- **Rate (INR)**: The shipping cost.
- **Validity**: `Effective From` date.
- **Version**: Grouping tag (e.g., `Band C`).

#### Add/Edit Rate

When you add or edit a rate, you can specify:

- **Location & Weight**: Select the zone and weight tier.
- **Rate**: Enter the cost in INR.
- **Validity**: Set start and end dates (`Effective From/To`).
- **Version**: Assign to a specific Rate Card Version.
- **Notes**: Internal comments.
- **Active**: Toggle availability.

### 3. Rate Card Versions

Organize your shipping rates into versioned sets using the **Rate Card Versions** page (`/admin-dashboard/manual-shipping/rate-card-versions`). This allows you to manage different pricing structures (e.g., "Holiday Rates" vs "Standard Rates") and switch between them.

#### Manage Versions

- **List View**: Displays all created versions with their **Effective Period** and **Rates Count**.
- **Actions**: Edit details or delete a version.

#### Add New Version

Click **+ Add Version** to create a container for your rates:

1.  **Version Name**: Unique identifier (e.g., `Band-C wef 10.02.2025`).
2.  **Description**: Optional context.
3.  **Effective Period**: Set the `From` and `To` dates. Leave "To" empty for ongoing validity.
4.  **Active**: **Important**: Create new versions as **Inactive** initially. Only activate them after you have uploaded the corresponding rates.

### 4. Country-Zone Management

The **Country-Zone Management** section (`/admin-dashboard/manual-shipping/countries`) is where you define the geographical logic for your rates.

#### Overview

- **Excel Upload**: Use the **Download Template** and **Upload Excel** buttons to bulk-import country-zone mappings.
  - **Required Columns**: `Country`, `Zone`.
  - **Optional**: `Country Code` (ISO 2-letter).
  - _Example_: Map "United States" to "Latin America" zone, "Australia" to "Asia".

#### List View

View the current mappings:

- **Columns**: Country, Code (e.g., `DZ`), Zone (e.g., `Africa`), and the count of active Rates for that zone.
- **Search & Filters**: Find specific countries or filter by Zone to check coverage.

### 5. Manage Locations

The **Locations** module (`/admin-dashboard/manual-shipping/locations`) handles the granular hierarchy of your shipping destinations.

#### Overview

- **Location Types**: Organize by structure (e.g., `Country` inside `Continent` inside `World`).
- **Filters**:
  - **Fallback**: Filter locations marked as "Fallback" (used when no specific rate is found).
  - **Status** and **Location Type**.

#### Add Location

- **Details**: Name, ISO Code, and Parent Location.
- **Fallback**: Check the **Mark as Fallback Location** box if this location should serve as a default for its parent region.

### 6. Exchange Rate Management

Control currency conversions (`/admin-dashboard/manual-shipping/exchange-rate`) to ensure accurate USD-to-INR shipping calculations.

#### Components

1.  **Default Fallback**: A mandatory safety net rate used if no specific date-matched rate exists. This cannot be deactivated.
2.  **Current Rate**: The large display shows the active rate for today (e.g., `1 USD = 100.00 INR`).
3.  **Add Rate**:
    - **Source**: Select `Bank Rate` or `Manual Entry`.
    - **Validity**: Set `Effective From` and optional `Effective To` dates.
4.  **Rate History**:
    - Manage past and future rates.
    - **Actions**: `Set Default` (make this the fallback), `Edit`, or `Disable`.

## Payments & Transactions

The **Transactions** module ensures full financial visibility, from individual payment logs to aggregate reports.

### All Transactions

Monitor every payment attempt (`/admin-dashboard/transactions/all`).

- **Stats Cards**: Real-time summary of **Total Revenue**, **Successful**, **Failed**, and **Pending** counts.
- **Filters**: Search by Order Code or Customer, and filter by Date Range.
- **Export**: Click the **Export** button to download the current view as a CSV file.

### Transaction Details

Clicking **View Details** reveals technical payment metadata, essential for debugging.

- **Overview**: Gateway Reference ID (**GID**), Provider (e.g., `payglocal`), and Payment Method.
- **Raw Webhook Data**: A JSON view of the exact response from the payment provider.
- **Timeline**: Precise timestamps for when the transaction was **Created** and **Last Updated**.

### Reports

The **Transaction Reports** page (`/admin-dashboard/transactions/reports`) visualizes your financial health.

- **Key Metrics**: Success Rate and Avg. Transaction value.
- **Visualizations**: Daily Revenue, Status Distribution, Payment Methods overlap, and Monthly Trends.

## Loyalty Program

The **Loyalty Program** (`/admin-dashboard/loyalty`) allows you to incentivize repeat purchases by rewarding customers with points.

### Configuration

Manage your points system under the **Settings** tab:

- **Points Per Dollar (Earning Rate)**: Define how many points a customer earns for every $1 spent (e.g., `1`).
- **Points to Dollar Ratio (Redemption Rate)**: Define the value of points when redeeming (e.g., `20` points = $1).
- **Minimum Redemption Threshold**: Set the minimum points required before a customer can redeem them (e.g., `200`).
- **Enable Loyalty Program**: Master toggle to turn the entire program on or off.

### Campaigns

Use the **Campaigns** tab to set up special earning events (e.g., "Double Points Weekend").
